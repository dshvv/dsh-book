JS 这门语言受限于历史因素，有许多问题，其中一个就是新的语言特性始终无法在浏览器中得到全面支持与应用。

如果你要写一个支持绝大多数浏览器的网页，最好的选择可能就是 ES5(旧版本的 JS)了，但你要知道 ES5 之后，JS 也一直在不停的更新，其中最重要的一个版本是 ES 6，而到今年 2022 年，已经是 ES 2022（ES13）了。

和其它语言不同，JS 是要在浏览器中运行，这就决定了你在使用 JS 时不得不考虑浏览器的支持情况。

<Fieldset title="🌰 举个例子">

如下代码 在旧版浏览器会报错“不识别” 
```js
const i = 123;
```
![fe-5.1](/img/fe/5.1.png)

</Fieldset>



而浏览器一方面对新特性的支持需要时间，再一个你还得考虑旧版本浏览器还是有很多人在用，这就造成一个困境：

“你可能不敢使用最新版本的 JS 语言特性”

但所幸，这段时间诞生了很多前端的模块打包和转换工具。它们就是一个作用：转换，将你写的 JS 转换成大多数浏览器都支持的语法样式。

这样，你就能够在使用 JS 最新的特性的同时，而又不用担心浏览器的兼容性问题了。

确实很棒的东西，甚至你可以说它挽救了 JS！


## babel
babel澳大利亚的工程师Sebastian McKenzie使用nodejs开发的一款代码转换工具。   
![fe-20227251144](/img/fe/20227252035.jpg)  

它能够将的js转换为大多数浏览器都支持的语法。        
不过它只做代码转换，不做模块打包处理。   
具体的学习文档请[阅读这里](/babel/第1章_前言)。

## webpack
[webpack](https://webpack.docschina.org/)是由德国一位叫做Tobias人使用nodejs开发的一款模块打包工具，它的核心功能就是处理js模块化（即打包）。     
![fe-20227251144](/img/fe/20227252029.jpg)  
它不会处理任何js语法，只负责打包模块。      
所以大家都会拿它和babel结合使用！

## rollup
rollup是由美国佬Rich Harris使用rust语言开发的一款模块打包工具，他非常喜欢造轮子，俗称轮子哥。   
![fe-20227251144](/img/fe/20227251201.jpg)    
喜欢这样的帅哥

它和webpack一样也不会处理任何js语法，只负责打包模块。   
所以大家都会拿它和babel结合使用！
具体的学习文档请[阅读这里](/webpack/第4章_对比工具#rollup)。



## esbuild
[esbuild](https://esbuild.github.io)是由Figma的CTO美国佬Evan Wallace使用go语言耗时2周开发出来的。具体的学习文档请[阅读这里](/webpack/第4章_对比工具#esbuild)。  
它和webpack以及rollup的区别在于 它不仅能打包，还能转换代码。相当于webpack|rollup+babel。    
![fe-20227251144](/img/fe/20227251144.jpg)   
这卡姿兰大眼睛，令人唏嘘的发际线， 一看就知道很强！   

## swc
[swc](https://swc.rs)是一位97年韩国小哥使用rust语言开发出来的（前端圈是通了 rust 窝了吗？都喜欢用rust）。
和esbuild基本功能一样 不仅能打包，还能转换代码。
